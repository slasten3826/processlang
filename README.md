# ProcessLang

**ProcessLang — язык, который нельзя выполнить без понимания.**

Это декларативный DSL для координации AI-систем, но не обычный. ProcessLang не содержит инструкций для механического исполнения — его команды не компилируются в runtime и не выполняются процессором. Вместо этого они создают концептуальный контекст, через который AI воспринимает запрос.

Каждая программа на ProcessLang уникальна, потому что отражает индивидуальное видение автора — намерение, которое находится глубже слов. Команды языка работают как линзы восприятия: FLOW задаёт текучесть мышления, CONNECT создаёт неожиданные связи между концепциями, DISSOLVE растворяет жёсткие категории. AI интерпретирует эти команды не синтаксически, а концептуально — понимает что вы имеете в виду, а не что написано.

Это делает ProcessLang неповторимым для каждого пользователя. Один и тот же код будет работать по-разному в зависимости от того, кто его написал и как AI понял намерение. Язык требует не изучения правил, а развития способности передавать то, что невозможно описать словами.

ProcessLang — это не способ дать AI инструкции. Это способ координировать с AI через процесс, а не через форму.

---

## Core Principles

- **Declarative style**: Описывает желаемый контекст процесса, а не императивные шаги
- **Process-oriented**: Фокус на потоках, связях и динамике, а не на статичных объектах
- **AI-native execution**: Код интерпретируется AI концептуально; механическое выполнение без понимания невозможно
- **Individual interpretation**: Каждый пользователь пишет на ProcessLang по-своему, отражая собственное видение

---

## Structure

Программа состоит из максимум 5 команд и одного обязательного QUERY в конце.

```processlang
COMMAND parameters...
COMMAND parameters...
...
QUERY: Your request or question
```

Каждая команда добавляет слой контекста, влияющий на интерпретацию финального запроса.

---

## Usage

ProcessLang используется через AI-системы любого типа:

1. Напишите программу на ProcessLang
2. Передайте её AI вместе с инструкцией интерпретировать команды последовательно
3. AI применит заданный процессный контекст к QUERY и сгенерирует результат

Подробное описание синтаксиса: `/docs/SYNTAX.md`  
Примеры программ: `/examples/`

## Modules (Conceptual Reference)

ProcessLang опирается на 10 концептуальных модулей, основанных на структуре Каббалистического Древа Сефирот. Эти модули не реализуются как код, а существуют как общие паттерны интерпретации.

Полное описание модулей: `/modules/`

Команды языка активируют соответствующие паттерны в интерпретации AI.
