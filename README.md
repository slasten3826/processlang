# ProcessLang

**A declarative DSL for coordinating AI interpretation through process-oriented context**

ProcessLang — доменно-специфический язык (DSL), предназначенный для задания контекста интерпретации запросов искусственным интеллектом. Он не предназначен для прямого выполнения традиционными runtime (CPU/GPU), а работает исключительно через концептуальную интерпретацию AI-системами любого типа (текстовые, мультимодальные и т.д.).

---

## Core Principles

- **Declarative style**: Описывает желаемый контекст процесса, а не императивные шаги.
- **Process-oriented**: Фокус на потоках, связях и динамике, а не на статичных объектах.
- **AI-native execution**: Код интерпретируется AI "на лету"; механическое выполнение без понимания невозможно.
- **Built-in anti-crystallization**: Намеренно неполная и концептуальная структура препятствует прямому копированию и догматическому использованию.

---

## Design Features

ProcessLang использует три механизма защиты от превращения в обычный фреймворк:

1. **Conceptual incompleteness** — ссылки на несуществующие в коде сущности, которые AI восстанавливает из контекста.
2. **Syntactic barriers** — усечённые конструкции и смешанная нотация (включая кириллицу), усложняющие механическое копирование.
3. **Interpretation dependency** — программа работает только при осмысленной интерпретации AI; слепое выполнение даёт бессмысленный результат.

Эти свойства не являются ошибками — они часть дизайна, обеспечивающая, что язык остаётся инструментом индивидуальной координации с AI, а не шаблоном для массового использования.

---

## Structure

Программа состоит из максимум 5 команд и одного обязательного QUERY в конце.

```processlang
COMMAND parameters...
COMMAND parameters...
...
QUERY: Your request or question
```

Каждая команда добавляет слой контекста, влияющий на интерпретацию финального запроса.

---

## Usage

ProcessLang используется через AI-системы любого типа:

1. Напишите программу на ProcessLang.
2. Передайте её AI вместе с инструкцией интерпретировать команды последовательно.
3. AI применит заданный процессный контекст к QUERY и сгенерирует результат.

Подробное описание синтаксиса: `/docs/SYNTAX.md`
