# ProcessLang Quick Start

Быстрое введение в использование ProcessLang.

---

## Как использовать ProcessLang

ProcessLang работает **только через компилятор**. Команды без модулей не имеют концептуального контекста и не дадут корректного результата.

### Требования

- Python 3.6+
- Git (для клонирования репозитория)

### Установка

```bash
git clone https://github.com/slasten3826/processlang.git
cd processlang
```

### Использование

```bash
# Компиляция программы
python compiler.py examples/simple_example.txt output.txt

# Скомпилированный prompt в output.txt содержит:
# - Необходимые модули с концептуальным контекстом
# - Ваши команды
# 
# Отправьте содержимое output.txt в AI-систему
```

### Что делает компилятор?

1. Читает вашу программу (.txt)
2. Определяет какие модули нужны
3. Загружает модули из `/modules/`
4. Собирает: модули + ваши команды
5. Сохраняет финальный prompt в выходной файл

**Структура после компиляции:**

```
[Содержимое модуля FluidCore.txt]
[Содержимое модуля EmergentConnection.txt]
...
---
FLOW engagement=0.9 resistance=0.1
CONNECT depth=0.8 between=["A", "B"]
QUERY: Your question
```

## Первая программа

### Пример: Простой запрос

```
FLOW engagement=0.7 resistance=0.3
QUERY: Объясни квантовую запутанность простыми словами
```

**Что происходит:**
- `FLOW` устанавливает уровень текучести мышления (0.7 = высокий, но не максимальный)
- `resistance=0.3` = умеренное сопротивление (сохраняем научную точность)
- AI интерпретирует запрос через этот контекст

### Пример: Междоменная связь

```
FLOW engagement=0.9 resistance=0.1
CONNECT depth=0.85 between=["музыка", "математика", "эмоции"]
DISSOLVE rigidity=0.7 target="жёсткие_категории"
QUERY: Как эти три домена взаимодействуют при создании музыки?
```

**Что происходит:**
- Максимальная текучесть (0.9)
- Глубокие связи между тремя доменами (0.85)
- Растворение жёстких категорий (0.7)
- AI ищет неочевидные паттерны взаимодействия

## Структура программы

```
COMMAND param1=value1 param2=value2
COMMAND param1=value1
COMMAND param1=value1
...
QUERY: Ваш вопрос или запрос
```

**Правила:**
- Максимум 5 команд (но можно меньше!)
- QUERY всегда последний
- Порядок команд имеет значение
- Пустые строки игнорируются
- Комментарии начинаются с `#`

## Базовые команды

### FLOW
Устанавливает базовый режим работы.

```
FLOW engagement=0.9 resistance=0.1
# Высокая вовлечённость, низкое сопротивление = творческий подход
```

### CONNECT
Создаёт связи между концепциями.

```
CONNECT depth=0.8 between=["A", "B", "C"]
# Глубокие связи между тремя концепциями
```

### DISSOLVE
Растворяет жёсткие рамки.

```
DISSOLVE rigidity=0.7 target="стереотипы"
# Ослабляет влияние стереотипов
```

### MANIFEST
Определяет формат вывода.

```
MANIFEST format="detailed" loss=0.1
# Детальный ответ с минимальной потерей информации
```

Полный список команд: `/docs/SYNTAX.md`

## Типичные паттерны

### Деконструкция-Реконструкция

```
DISSOLVE rigidity=0.8 target="традиционные_определения"
CONNECT depth=0.9 between=["новая_концепция_1", "новая_концепция_2"]
ENCODE type="network" loss=0.3
```

Сначала разрушаем старые рамки, затем создаём новые связи, затем структурируем.

### Глубокий анализ

```
FLOW engagement=0.95 resistance=0.05
OBSERVE distance=0.3 target="предположения"
CYCLE mode="intense" intensity=0.8
```

Максимальная текучесть + самонаблюдение + итеративное углубление.

### Минималистичный подход

```
FLOW engagement=0.7 resistance=0.3
QUERY: Ваш вопрос
```

Не обязательно использовать все 5 команд — используйте столько, сколько нужно.

## Советы

### Начинающим

1. **Начните с простого**: `FLOW` + `QUERY`
2. **Добавляйте команды постепенно** по мере понимания
3. **Изучите примеры**: `/examples/` содержит реальные программы
4. **Экспериментируйте с параметрами**: меняйте значения и смотрите на результат

### Продвинутым

1. **Порядок команд важен**: каждая команда строится на предыдущей
2. **Комбинируйте команды осмысленно**: избегайте противоречий
3. **Не все задачи требуют 5 команд**: минимализм часто эффективнее
4. **Используйте комментарии**: особенно в сложных программах

### Для всех

- **ProcessLang работает через понимание**, а не механическое выполнение
- **Не копируйте слепо** — адаптируйте под свою задачу
- **Язык намеренно концептуален** — это не баг, это фича
- **AI-системы интерпретируют по-разному** — это нормально

## Отладка

### Программа не даёт ожидаемого результата?

1. **Проверьте порядок команд**: логичен ли переход от одной к другой?
2. **Проверьте значения параметров**: может `engagement` слишком низкий?
3. **Упростите программу**: уберите команды по одной и найдите проблему
4. **Попробуйте другую AI-систему**: разные модели интерпретируют по-разному

### Компилятор выдаёт ошибку?

```bash
# Проверьте синтаксис команд
# Убедитесь что файлы модулей существуют
ls -la modules/

# Проверьте что программа читается
cat examples/your_program.txt

# Используйте verbose режим (если добавлен)
python compiler.py -v examples/your_program.txt
```
