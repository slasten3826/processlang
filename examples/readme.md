# ProcessLang Examples

Коллекция примеров программ на ProcessLang, демонстрирующих различные подходы к координации AI через процесс-ориентированный контекст.

---

## Категории

### Философские и концептуальные

**karma.txt**  
Понимание кармы как вечного процесса через призму каббалистических концепций (Chesed, Gevurah, Netzach). Демонстрирует растворение жёстких определений и циклическое мышление.

**eternal_flow.txt**  
Визуализация процесса как первичной реальности. Пример использования ProcessLang для генерации изображений, фокус на текучести и отсутствии фиксированных форм.

### Метакогнитивные

**learning.txt**  
Глубокое обучение как процесс трансформации, а не аккумуляции. Работа с confusion, растворение старых рамок, усиление новых паттернов через RUNTIME.

**creative_block.txt**  
Растворение творческих блоков через увеличение потока и снижение требований к результату. Демонстрирует работу с сопротивлением.

**decision.txt**  
Принятие решений как коллапс пространства возможностей. Использование команды CHOOSE и работа с непройденными путями.

### Аналитические

**geopolitics.txt**  
Структурный анализ устойчивости ЕС без идеологических нарративов. Минималистичная программа с высокой эффективностью.

**демография.txt**  
Анализ демографических факторов через структурные паттерны и исторические прецеденты. Растворение идеологических рамок для объективного анализа.

**учебный_план.txt**  
Разработка 72-часовой образовательной программы по ООП через Pygame. Демонстрирует соединение множественных доменов (когнитивное развитие, игровая механика, математика, архитектура).

### Технические

**RAG_в_продакшене.txt**  
Ультра-минималистичный пример для технических запросов. Показывает что ProcessLang не требует всех 5 команд - используйте столько, сколько нужно.

**simple_example.txt**  
Базовый пример для тестирования. Демонстрирует структуру программы и базовый поток команд.

### Практические

**garden_path.txt**  
Инженерная задача: оптимальное строительство садовой дорожки с ограниченными ресурсами. Пример применения ProcessLang к конкретным практическим задачам.

### Визуальные

**dawn_transformation.txt**  
Генерация изображения рассвета как момента трансформации. Демонстрирует использование ProcessLang для AI-генерации изображений через промпт-инжиниринг.

**eternal_flow.txt**  
Абстрактная визуализация вечного процесса. Растворение границ, циклическое движение, эмерджентная сложность.

## Рекомендации по использованию

1. **Начните с простых примеров**: `simple_example.txt` → `creative_block.txt` → `karma.txt`

2. **Для своих задач используйте релевантную категорию**:
   - Философский вопрос → `karma.txt` как шаблон
   - Технический запрос → `RAG_в_продакшене.txt` как шаблон
   - Визуальная задача → `dawn_transformation.txt` как шаблон

3. **Обратите внимание на паттерны**:
   - `DISSOLVE` + `CONNECT` = деконструкция-реконструкция
   - `OBSERVE` + `CYCLE` = метакогнитивное углубление
   - `FLOW` + `MANIFEST` = базовый поток с финализацией

4. **Комментарии необязательны** в продакшене, но полезны при обучении.

5. **Не копируйте слепо** — адаптируйте под свою задачу. ProcessLang работает через понимание, а не через шаблоны.

## Экспериментируйте

Эти примеры показывают возможности ProcessLang, но не ограничивают их. Язык намеренно гибкий и концептуальный.

Создавайте свои программы, комбинируйте команды по-новому, находите паттерны которых нет в примерах.

ProcessLang — это инструмент координации с AI, а не набор готовых решений.
